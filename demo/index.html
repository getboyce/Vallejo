Vallejo Design System 1.2

Vallejo is the system Copart builds on. It is Layer 2 — the how that executes the Copart Design Vault, the why behind every decision. The Vault sets the principles; Vallejo ships the tokens, type, and components that carry them. When the two ever conflict, the Vault wins.

Precision

The warmth of competence. Obsessive alignment, whitespace, and typographic discipline — every value from a token, every line-height a multiple of 4. When members stake five figures on vehicles they have never touched, precision is the trust signal.

Institutional confidence

The quiet authority of a platform that has moved millions of vehicles. Restraint over decoration. One primary action per view. No hedging, no exclamation marks. Confidence is also what we choose not to show.

Vitality

Auctions are alive — competitive, time-bound, consequential. Motion confirms action; status stays visible; urgency is real, never manufactured. The pulse serves information, never decoration.

The design filter

Apply to every screen, component, and word: Does this build trust? Does it respect expertise? Does it read at multiple depths? If the answer to any is no, the design is wrong.

Foundations

The primitives every component resolves to: color, type, spacing, radius, elevation, and motion. Each value is a token in dist/tokens.css — nothing in the system is hand-picked.

Build with Vallejo

Two stylesheets and an unprefixed class API. Drop them in and you are building exact, on-brand Copart prototypes — no framework, no build step, no Tailwind.

The two-stylesheet how-to

Link tokens first, then the component layer. The component classes consume var(--…) tokens, so the order matters.

In your <head>
<link rel="stylesheet" href="/dist/tokens.css">
<link rel="stylesheet" href="/dist/vallejo.css">

The class API

Base plus modifiers, unprefixed: .btn .btn-primary .btn-md. State lives in native CSS and ARIA — :hover, :focus-visible, [aria-pressed], [aria-selected] — never an .is-selected class. Color, space, and radius always resolve to a token; never hardcode a value.

Do

Compose from the shipped classes and let ARIA carry state.

<button class="btn btn-primary btn-md">Place bid</button>
Don't

Invent state classes or hardcode a hex the token already owns.

<button class="btn is-active" style="background:#2059CC">
