Foundations
Design Philosophy
Every pixel in the Copart experience should earn trust, reduce friction, and move the user closer to the vehicle they need. We design for clarity first, beauty second. Because when millions of dollars flow through an auction platform, confidence is the most valuable feature we ship.
Core Principles
1

Clarity Builds Trust

Every element must communicate its purpose immediately. Users are making significant financial decisions — ambiguity erodes confidence.

Do: A bid button reads "Place Bid" with a confirmation step.
Don't: A button reads "Submit" with no indication of commitment.
2

Progressive Disclosure

Show only what's needed at each step. Layer information so newcomers aren't overwhelmed and power users aren't slowed down.

Do: Vehicle search shows key specs in the card; full details on the detail page.
Don't: A search results card showing 20+ data fields in 10pt type.
3

Accessible by Default

Every user deserves the same auction experience. WCAG 2.1 AA as a floor, not a ceiling.

Do: An auction status badge uses color AND icon plus label.
Don't: A green dot with no label to indicate a live auction.

When to Animate
Do Animate
Action confirmation — Heart fill, checkmark draw, success toast
State changes — Toggle slide, dropdown open, modal enter
Attention direction — Bid price pulse, badge count update, new item appear
Loading feedback — Skeleton shimmer, spinner, progress bar fill
Spatial context — Page transitions, tab switches, card grid stagger
Don't Animate
Decoration — Bouncing logos, floating elements, ambient motion
Delays — Never make users wait for animation to complete before acting
Scrolljacking — Don't override native scroll or parallax on content pages
Repetitive loops — No infinite animations except spinners and skeleton
Form interactions — Typing, selecting, tabbing between fields must be instant
