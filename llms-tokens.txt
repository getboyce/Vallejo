# Vallejo Tokens — full reference

Every value in Vallejo. Components consume `var(--token)` and NEVER hardcode hex/size.
Bind the SEMANTIC alias (text/*, surface/*, border/*, interactive/*, status roles) — never a primitive,
never a raw hex. Reach for a primitive ramp step only when no semantic alias fits.

Version + date live ONLY in version.json (1.2.0 / 2026-06-20). Source of truth = Figma "Vallejo DS - 1.2"
(fileKey bSOhenbjW7fx7CvGhOXfC0). CSS var names are unprefixed-kebab: `--gray-600`, `--space-4`, `--radius-xl`, etc.

Canonical naming:
- `gray` is the canonical neutral ramp (gray-0 = white .. gray-950). `neutral-*` is a DEPRECATED alias only → resolves to `gray-*`.
- `layer-*` is canonical for z-index. `z-*` is a DEPRECATED alias only.
- NO orange ramp (removed). `gold #FA9252` exists for Premier rings ONLY. Urgency uses the warning/error ramps.
- Info is RETIRED → use blue (informational = blue's job). There is no `info-*` ramp.
- Error/destructive uses the error ramp — never a raw red.

Color governance (binding):
- NEVER derive a color with opacity (no `rgba(ink, .07)`). Separation = named SOLID tokens; hairlines are gray-200 SOLID.
  Shadows and scrims are the ONLY sanctioned alpha exemption — they model light, not color.
- State is NEVER color-alone (CVD): chips carry a word; LIVE keeps the ● dot. success-600 and error-600 are
  near-isoluminant under deuteranopia.
- `text-display-muted` (gray-500) is ≥24px display text only. `text-on-header-muted` (gray-400) is dark-bars only
  (passes 4.77:1 on dark bars; FAILS on shelf/field gray-700).
- `border-control` (gray-500) = inputs only (an empty input's boundary is its identifier → needs 3:1).
  Labeled controls incl. Secondary buttons use `border-default` (gray-300) per W3C 1.4.11.
- Contrast spine on white: 50→1.075 · 100→1.16 · 200→1.37 · 300→1.78 · 400→2.62 · 500→3.64 · 600→5.75 · 700→8.6 · 800→12.6 · 900→18.1.
  Any 600 is button-grade; any 800-on-100 is a legible tonal badge.

## 3a. Breakpoints (responsive grid anchors)

Mobile-first `min-width`. `desktop-lg` (1440) is the canonical design width. The 12-column grid collapses to
8 then 4; content caps at 1280px max-width (margins grow past 1440). Grid: 12/8/4 columns · gutter 24/16/16 ·
margin 80/32/16 (desktop/tablet/mobile). Every measurement stays on the space scale.
| Breakpoint | Min-width | Columns |
|---|---|---|
| mobile-sm   | 320  | 4  |
| mobile      | 375  | 4  |
| tablet      | 768  | 8  |
| desktop-sm  | 1024 | 12 |
| desktop     | 1280 | 12 |
| desktop-lg  | 1440 | 12 |  (canonical design width)
| desktop-xl  | 1920 | 12 |
Safe areas: respect env(safe-area-inset-*) on notched/Dynamic Island devices (bottom nav clears inset-bottom + 8px;
fullscreen modals honor all four insets); set viewport meta `viewport-fit=cover`. Full grid + safe-area prose in DESIGN.md §2.4.

## 7. Type scale (style → family / weight / size / line-height)

Bind the named style, never raw font props. Primary font Inter (UI + reading); data font JetBrains Mono 500
(numbers you compare, codes, timers, VINs). Mono never in labels/prose. Weights 400/500/600/700 only.
Every line-height is a multiple of 4. Off-grid sizes (10/17/26/30) banned. LS = letter-spacing.

**Apply type with the `.text-<style>` utility classes in `dist/vallejo.css`** — never hand-write `font-size`/`line-height`. Use them on page chrome AND inside markup: `.text-display-large` · `.text-display-small` · `.text-headline` · `.text-title-1` · `.text-title-2` · `.text-title-3` · `.text-body-large` · `.text-body` · `.text-body-emphasis` · `.text-list-title` · `.text-subheadline` · `.text-body-small` · `.text-footnote` · `.text-caption` · `.text-overline` · and mono `.text-mono-large` · `.text-mono-medium` · `.text-mono-small`. ## 8. Dark mode (planned — NOT shipped yet)

The build is LIGHT-ONLY today. Dark mode is planned but not shipped yet — Figma's dark values are placeholders
("light-only; do not ship dark"), so the values shown in §2 are the only resolutions that exist right now. Do
NOT rely on dark resolving in this build.

The PLANNED mechanism is a `data-theme="dark"` layer: set `data-theme="dark"` on the root (or any subtree) and the
SAME semantic aliases you bind in §2 will resolve to curated dark ramp values instead of the light ones — you would
NOT change the token names in your markup. An ADA dark guardrail will preserve contrast targets in dark, and dark
chrome (header/footer/shelf) is identical in both themes. This will land in a dedicated pass once Figma finalizes
dark. Until then, bind semantic aliases (not primitives) so a future single `data-theme=dark` switch can re-theme a
prototype — but treat the system as light-only for now.
