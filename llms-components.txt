# Vallejo 1.2 — Components (full per-component reference)

The single home for component DETAIL. ~50 components, grouped by Figma page. For each:
variant axes + total count · states · anatomy + key rules · a11y notes · a canonical copy-paste HTML snippet.

Read order: STEP 0 = the Copart Design Vault (github.com/getboyce/Copart-Design-Vault) — the *why* (Layer 1). Vallejo is the *how* (Layer 2). When they conflict, the Vault wins. STEP 1 = llms-tokens.txt for the token values these snippets reference. This file is Layer 2 detail; DESIGN.md is the map.

## How to read the snippets (the class API — DEC-12)
- **Unprefixed classes**: base + modifiers. `.btn` / `.btn-primary` / `.btn-md`; `.card`; `.badge` / `.badge-success`; `.chip`; `.modal`. Phase 2 `dist/vallejo.css` implements exactly these classes — they are the contract.
- **States via NATIVE CSS only** — `:hover`, `:focus-visible`, `[disabled]` / `[aria-disabled]`, `[aria-pressed]`, `[aria-selected]`, `[aria-current]`, `aria-invalid`, `[open]`. **Never** a `.is-hover` / `.selected` state class.
- **Tokens, never literals.** Every color/size/radius/space resolves to a `var(--token)`. Snippets name the token in a comment where it is load-bearing; never hardcode a hex or px that a token owns. Values live in llms-tokens.txt.
- **Selected register** (one uniform across chips, selected rows, selected options, active filter chip): `--blue-50` wash · `--blue-300` border · `--blue-800` label. Memorize it once. (Watchlist saved is the exception — filled red-500 heart on a neutral surface, never the blue register.)
- **Focus** is a real state on every interactive component: a **solid 2px ring**, never a shadow glow. `--border-focus` (blue/600) on light surfaces; `--border-focus-on-dark` (blue/300) on dark chrome. Ring radius is concentric (control radius + 3). Engineer it with `outline: 2px solid var(--border-focus); outline-offset: 2px;`.
- **Buttons are raised** via the named Elevation effect styles. **Primary + Destructive are RAISED:** a vertical gradient fill (`var(--interactive-primary)` → `var(--interactive-primary-grad-end)`; Destructive uses `var(--interactive-destructive)` → `var(--interactive-destructive-grad-end)`) plus `box-shadow: var(--elevation-primary-rest)` / `var(--elevation-destructive-rest)` (hover/pressed variants swap the rest token). **Secondary, Quiet, and Chips** wear the raised KEY edge (`var(--elevation-key)`); **Quiet is the flattest.** Floating surfaces (Menu/Popover/Toast/Modal/Shelf/Tooltip, the floating Watchlist heart) use `var(--elevation-overlay)` / `var(--elevation-modal)`. Focus is still a solid 2px ring, never a glow.
- **44px hit-area floor** regardless of visual size (a 32px control pads out to 44px in code).
- **Voice in every label**: sentence case, verb-first, no prices in button labels, no emoji. Numbers you compare (VIN, lot #, bid, timestamp) render in JetBrains Mono.

This file documents the ~50 Figma components ONLY. There is **no** Header, Sidebar, Link, Chart, Image, Video, or Date Picker component (dropped — DEC-4). Do not reference them as components.

# Buttons

## Button
Figma: `Button 1.2` · node `4079:255` · set · 300 variants.

- **Variant axes** (300 = 4×5×3×5):
  - **Type** (4): Primary · Secondary · Quiet · Destructive. *(There is NO Tertiary — that's a legacy misname for Quiet — and NO Success button.)*
  - **Size** (5): XS · SM · MD · LG · XL → heights 28 / 32 / 40 / 48 / 56 (the 8px rhythm). **40 (MD) is the lingua franca.** XS is for dense table rows only; XL for marketing heroes; LG+ for comfortable touch.
  - **Icon** (3): None · Leading · Trailing.
  - **State** (5): Default · Hover · Active · **Focus** · Disabled.
- **States:** Default · Hover · Active · Focus · Disabled.
- **The one rule:** Primary + Destructive are **raised** (gradient fill + layered `--elevation-primary-*` / `--elevation-destructive-*` shadows); Secondary/Quiet wear the raised KEY edge (`--elevation-key`), Quiet flattest. Primary is reserved for the single most important action on the screen (bid, pay, confirm) — **one Primary per view**.
- **Anatomy + key rules:**
  - Fill + elevation by type: **Primary** is RAISED — a vertical gradient `var(--interactive-primary)` (blue/600) → `var(--interactive-primary-grad-end)` (#1D50B8) + `box-shadow: var(--elevation-primary-rest)` (hover/pressed swap to the hover/pressed elevation tokens); the fill ladder still deepens 600 → 700 (hover) → 800 (active) under the gradient. **Destructive** is RAISED the same way — gradient `var(--interactive-destructive)` (error/600) → `var(--interactive-destructive-grad-end)` (#AF001F) + `var(--elevation-destructive-rest)`; ladder error 600 → 700 → 800. **Secondary** is fully neutral: white fill, ink label, refined-light border (NOT a brand-color outline — that's a legacy pattern), wearing the raised KEY edge (`var(--elevation-key)`). **Quiet** is text-only blue label, no fill/border at rest — the flattest type.
  - **Refined-light border** (Secondary/labeled): rest `--border-default` (gray/300) → hover gray/400 + `--surface-canvas` (gray/50) fill → pressed gray/500 + gray/100 fill. The line advances on approach. (A labeled button is identified by its label per W3C 1.4.11, so its boundary is decorative — this is why it may be lighter than the 3:1 an empty input needs.)
  - **Radius scales with height:** SM 6 (`--radius-md`) · MD/LG 8 (`--radius-lg`) · XL 12 (`--radius-xl`). XS shares 6.
  - **Optical padding** (icon side pulls tighter than text side): text-side 12/16/20, icon-side 8/8/12 (XL 16); icon↔label gap 6/8/8.
  - **Labels** Inter Medium 500, sizes 12/14/14/16/18 across XS–XL.
  - **Disabled** is a multi-cue shift (not color alone): ink 16:1 → gray/400 2.65:1 + border drops to gray/200 + no hover response. Luminance-encoded, so it survives all color-blindness. Contrast-exempt by spec (inactive component).
  - Blue belongs ONLY to: Primary fills, Quiet labels, links, focus. Loading state is component-pass debt (pending).
- **a11y:** `<button>` (or `role="button"`). Icon-only requires an `aria-label`. Loading = `aria-busy="true"` + accessible "Loading, please wait". Use `aria-disabled="true"` (not the `disabled` attribute) when focus order must be preserved. 44×44 hit-area floor. Focus = solid 2px blue/600 ring (blue/300 on dark), concentric.

```html
<!-- Primary (MD, the default) — one per view; verb-first label, no price -->
<button class="btn btn-primary btn-md">Place bid</button>

<!-- Secondary (MD) — neutral ink label + refined-light border -->
<button class="btn btn-secondary btn-md">Cancel</button>

<!-- Quiet (MD) — text-only blue, no fill/border at rest -->
<button class="btn btn-quiet btn-md">View details</button>

<!-- Destructive (MD) — error fill ladder; reserve for dangerous commits -->
<button class="btn btn-destructive btn-md">Delete vehicle</button>

<!-- Sizes: XS (dense table rows) and LG (comfortable touch) -->
<button class="btn btn-primary btn-xs">Bid</button>
<button class="btn btn-primary btn-lg">Pay now</button>

<!-- Icon-leading — icon is decorative, label carries meaning -->
<button class="btn btn-secondary btn-md">
  <svg class="btn-icon" aria-hidden="true"><use href="#icon-filter"></use></svg>
  All filters
</button>

<!-- Icon-only — MUST name the action + carry .btn-icon-only (collapses to a square) -->
<button class="btn btn-quiet btn-md btn-icon-only" aria-label="More actions">
  <svg class="btn-icon" aria-hidden="true"><use href="#icon-more-vertical"></use></svg>
</button>

<!-- Disabled (preserve focus order) -->
<button class="btn btn-primary btn-md" aria-disabled="true">Place bid</button>

<!-- Loading -->
<button class="btn btn-primary btn-md" aria-busy="true" aria-label="Placing bid, please wait">
  <span class="spinner spinner-16 spinner-inverse" aria-hidden="true"></span>
  Placing bid
</button>
```
```css
/* Focus is a state — native, solid ring, no glow */
.btn:focus-visible { outline: 2px solid var(--border-focus); outline-offset: 2px; }
```

## Watchlist
Figma: `Watchlist 1.2` · node `4098:509` · set · 32 variants.

- **Variant axes** (32 = 2×2×4×2): **Button type** (2): Full · Compact · **Saved** (2): Off · On · **State** (4): Default · Hover · Active · Disabled · **Size** (2): Medium · Small (Small = 32px circle that floats over lot photography).
- **States:** Default · Hover · Active · Disabled.
- **The one rule:** Heart fills **red (red-500)** when active — the one favorite red that is NOT a destructive cue. Saved flips the label **Watchlist → Watching** and the heart from outline to filled, never color alone.
- **Anatomy + key rules:** Save-to-list toggle (bookmark pattern). Unsaved wears the raised-key edge (matches Secondary). Saved/On stays on a **neutral surface** — state is carried by the **filled red-500 heart** + the word "Watching," not a blue register and not color alone. Hearts: `icon.heart` (outline) → `icon.heart-filled`, both painting `--interactive-favorite` (red-500) via currentColor. Small 32px is floating by construction, so it is the documented elevation exception: it carries Elevation/Hover + a hairline. Heart inset is 12 (wider than the button icon-side 8) — heart-specific optical value.
- **a11y:** `aria-pressed` true/false carries saved state. Visual 32px (Small) but **required 44px expanded hit area in code**. State is the label flip + filled red heart, never color alone.

  Watching

<!-- Small (over photography) — visual 32px, 44px hit area in code -->
<button class="watchlist watchlist-small" aria-pressed="false" aria-label="Add to watchlist">

# Inputs

## Text Field
Figma: `Input / Text Field 1.2` · node `4113:79` · set · 18 variants.

- **Variant axes** (18 = 3×6): **Size** (3): SM · MD · LG · **State** (6): Default · Hover · Filled · Error · Success · Disabled.
- **States:** Default · Hover · Filled · Error · Success · Disabled.
- **The one rule:** Error owns hue / focus owns weight. Border is **refined-light**, advancing rest gray/300 → hover gray/400 → focus blue 2px.
- **Anatomy + key rules:** Label above (14/500). Helper below (13). "(Optional)" is plain text, **never a required star**. INPUTS differ from buttons: an empty field's boundary IS its identifier, so rest border = `--border-control` (gray/500, 3.65:1) — heavier than a labeled button's gray/300. Placeholder gray, value ink. VIN / lot # / amount values render in mono. Search is its own component (see Search).
- **a11y:** Visible `<label for>` required (labels are not placeholders). `aria-invalid="true"` on error; error text wired via `aria-describedby`. `aria-required="true"` for mandatory fields.

```html
<!-- Default (MD) -->
<div class="field">
  <label class="field-label" for="email">Email</label>
  <input class="input input-md" id="email" type="email" placeholder="name@example.com">
  <p class="field-helper" id="email-help">We'll send your receipt here.</p>
</div>

<!-- Error — hue + describedby; role on the message -->
<div class="field">
  <label class="field-label" for="vin">VIN</label>
  <input class="input input-md input-mono" id="vin" value="1HGES16503"
         aria-invalid="true" aria-describedby="vin-err">
  <p class="field-error" id="vin-err">VIN must be 17 characters — check for a typo.</p>
</div>

<!-- Optional (plain text, no star) + Success + Disabled -->
<div class="field">
  <label class="field-label" for="nick">Nickname <span class="field-optional">(Optional)</span></label>
  <input class="input input-md" id="nick">
</div>
<input class="input input-sm" value="Saved" aria-describedby="ok"><span id="ok" class="field-success">Looks good.</span>
<input class="input input-md" value="Locked" disabled>
```
```css
.input:focus-visible { outline: 2px solid var(--border-focus); outline-offset: 2px; }
```

## Slider
Figma: `Input / Slider 1.2` · node `4163:228` · set · 4 variants.

- **Variant axes** (4 = 2×2): **Type** (2): Single · Range · **State** (2): Default · Hover.
- **States:** Default · Hover.
- **The one rule:** Recessed track + raised-key thumb — the same rise/recess language as the Toggle. (The Figma set documents a Range variant, but the shipped CSS is single-thumb native; dual-handle needs JS.)
- **Anatomy + key rules:** Track 6px (`--surface-sunken` well) + blue/600 fill (via `accent-color`) + raised-key white thumb 18px. Thumb border gray/300 → gray/400 hover → blue/600 focus-ring. Disabled = gray/400 fill + gray/100 thumb, flat. **Single-thumb only** — a dual-handle range (min + max) is not expressible with one native `<input type="range">` and needs JS.

## Component → token cheat-sheet (bindings the snippets assume)
Phase 2 `dist/vallejo.css` resolves these. Values in llms-tokens.txt.

- **Selected register** (chips, selected rows/options, active filter chip): `--blue-50` wash · `--blue-300` border · `--blue-800` label. (Watchlist saved = filled red-500 heart on a neutral surface, not this register.)
- **Focus ring**: `--border-focus` (blue/600) light · `--border-focus-on-dark` (blue/300) dark chrome; `outline: 2px solid; outline-offset: 2px` (concentric +3 radius).
- **Refined-light button border** (Secondary): rest `--border-default` (gray/300) → hover gray/400 + `--surface-canvas` → pressed gray/500 + `--surface-sunken`.
- **Input boundary** (empty field IS the control): `--border-control` (gray/500, 3.65:1) — heavier than buttons.
- **Tonal recipe** (Badge Subtle, Alert): `--{tone}-100` fill + `--{tone}-800` content. **Solid** (Badge/Banner): `--{tone}-600` fill + white.
- **Quiet surface** (Lot/Stat card, List/Table Row): `--surface-raised` (white) + `--border-hairline` (gray/200), NO zebra, NO resting shadow.
- **Avatar membership ring** (`.avatar-basic` / `.avatar-premier`): a 2px ring + 2px transparent gap (`outline` + `outline-offset: 2px`) — Basic `--gray-300`, Premier `--gold` (#FDC146, = warning-200).
- **Radius family**: controls `--radius-lg` (8) · cards/menus/toasts/popovers `--radius-xl` (12) · modals/shelves `--radius-2xl` (16) · pills (badge/chip/tag) `--radius-pill` · tags/tooltips `--radius-md` (6).
- **Elevation** (floating surfaces only): Elevation/Hover (floating Watchlist), Elevation/Overlay (Menu/Popover/Toast/Tooltip/Select menu), Elevation/Modal (Modal/Shelf), Elevation/Key (raised segment/chip).
- **Layers**: dropdown 1000 · sticky 1100 · drawer 1300 · scrim 1400 · modal 1500 · toast 1600 · tooltip 1700 (no header layer — DEC-11).
- **Mono** (`.input-mono` / JetBrains Mono): VIN · lot # · bid · counter · timestamp. Hero singular numbers (Stat Card value) stay Inter + tabular.
